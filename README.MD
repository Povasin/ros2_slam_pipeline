# 🚀 Hilti-Trimble SLAM Challenge 2026 - Automated Pipeline

Полностью контейнеризированный пайплайн на базе **ROS 2 Jazzy** и **Docker** для решения задач визуально-инерциального SLAM (OpenVINS). 

Проект автоматически считывает сырые данные датчиков (`rosbag.db3`), строит 3D-карту помещений, извлекает и синхронизирует панорамные кадры с траекторией, а также визуализирует результат в собственном 3D-плеере (Open3D + PyQt5) и готовит ZIP-архив для сабмишена.

## ✨ Особенности
* 📦 **Zero-Setup:** Вся среда (ROS 2, OpenCV, Open3D, Ceres) упакована в Docker.
* 🤖 **Один скрипт для всего:** Команда `./start.sh` сама соберет образ, скомпилирует ROS-пакеты, запустит алгоритмы и выдаст готовый результат.
* 🗺️ **Интерактивный 3D Viewer:** Встроенный плеер для просмотра облака точек, траектории и синхронизированных кадров с камеры.
* 🪟 **Кроссплатформенность:** Автоматическая настройка графического сервера для отрисовки RViz и 3D-сцен в Ubuntu (X11) и Windows 11 (WSL2 + Docker Desktop).

## 🛠️ Установка и запуск

### 1. Требования
* Установленный **Docker** (и Docker Compose).
* Настроенный Git.
* Операционная система: Ubuntu (рекомендуется сессия Xorg) или Windows 11 (WSL2).

### 2. Клонирование репозитория
```bash
git clone https://github.com/Povasin/ros2_slam_pipeline.git
cd ros2_ws

### 3. Подготовка датасета
Тяжелые файлы .db3 игнорируются в Git. Перед запуском поместите данные от организаторов (rosbag) по следующему пути:
Plaintext
ros2_ws/
└── data/
    └── floor_1/
        └── 2025-05-05/
            └── run_1/
                └── rosbag/
                    └── rosbag.db3   <-- Поместите файл сюда

### 4. ЗАПУСК
./start.sh floor_1 2025-05-05 run_1

1)Что сделает скрипт:
2)Соберет Docker-образ (только при первом запуске).
3)Скомпилирует пакеты (colcon build).
4)Запустит сервер карты и SLAM (OpenVINS).
5)Проиграет rosbag и построит карту.
6)Распакует картинки и создаст sync_index.csv.
7)Откроет 3D-плеер для проверки.

📁 Структура выходных данных (Outputs)
После успешного завершения пайплайна, в вашем рабочем пространстве появятся следующие данные:

data/map_floor_1_2025-05-05_run_1.pcd — Итоговое 3D облако точек.

data/trajectory_floor_1_2025-05-05_run_1.txt — Файл траектории (Odometry).

dataset/floor_1/images/ — Распакованные кадры с камеры (.jpg).

dataset/floor_1/sync_index.csv — CSV индекс (связка картинок и координат).

🕹️ Использование 3D-Viewer'а
Встроенный slam_viewer.py открывается автоматически в конце работы скрипта.

Используйте ползунок внизу окна, чтобы перемещаться по времени.

Слева будет отображаться кадр с камеры для текущей временной метки.

В 3D-окне красная сфера будет двигаться по оранжевой траектории внутри облака точек.
