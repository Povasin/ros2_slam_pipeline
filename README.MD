Полностью контейнеризированный пайплайн на базе **ROS 2 Jazzy** и **Docker** для решения задач визуально-инерциального SLAM (OpenVINS). 

Проект автоматически считывает сырые данные датчиков (`rosbag.db3`), строит 3D-карту помещений, извлекает и синхронизирует панорамные кадры с траекторией, а также визуализирует результат в собственном 3D-плеере (Open3D + PyQt5) и готовит ZIP-архив для сабмишена.

---

## ✨ Особенности

* **📦 Zero-Setup:** Вся среда (ROS 2, OpenCV, Open3D, Ceres) упакована в Docker. Никаких конфликтов зависимостей на хост-машине.
* **🤖 Автоматизация полного цикла:** Команда `./start.sh` сама соберет образ, скомпилирует ROS-пакеты, запустит алгоритмы и выдаст готовый результат.
* **🗺️ Интерактивный 3D Viewer:** Встроенный кастомный плеер для просмотра облака точек, траектории и синхронизированных кадров с камеры.
* **🪟 Кроссплатформенность:** Автоматическая настройка графического сервера для отрисовки RViz и 3D-сцен в Ubuntu (X11) и Windows 11 (WSL2 + Docker Desktop).

## 🛠️ Установка и запуск

### 1. Требования
* Установленный **Docker** (и Docker Compose).
* Настроенный **Git**.
* Операционная система: **Ubuntu** (рекомендуется сессия Xorg) или **Windows 11** (WSL2).

### 2. Клонирование репозитория
```bash
git clone [https://github.com/Povasin/ros2-slam-pipeline.git](https://github.com/Povasin/ros2-slam-pipeline.git) test_ws
cd test_ws
```

### 3. Подготовка датасета
Поместите исходный файл `rosbag.db3` в структуру директорий следующим образом:
```text
ros2_ws/
└── data/
    └── floor_1/
        └── 2025-05-05/
            └── run_1/
                └── rosbag/
                    └── rosbag.db3   <-- Поместите файл сюда
```

### 4. Запуск пайплайна
```bash
chmod +x start.sh run_pipeline.sh
./start.sh floor_1 2025-05-05 run_1
```

## ⚙️ Конфигурация параметров
Настройки интринсиков и экстринсиков камеры, а также параметры алгоритма OpenVINS не захардкожены. Вы можете легко адаптировать пайплайн под другие датасеты или собственного робота, изменив YAML-файлы конфигурации:

* **Конфигурация SLAM:** `src/openvins_config/params/estimator_config.yaml`
* **Параметры сенсоров (Camera/IMU):** `src/openvins_config/params/sensor_calibration.yaml`

## 📁 Структура выходных данных (Outputs)
После успешного завершения пайплайна, в вашем рабочем пространстве появятся готовые артефакты:

| Путь к файлу / директории | Описание |
| :--- | :--- |
| `data/map_...pcd` | Итоговое 3D облако точек. |
| `data/trajectory_...txt` | Файл вычисленной траектории (Odometry). |
| `dataset/.../images/` | Директория с распакованными кадрами камеры (`.jpg`). |
| `dataset/.../sync_index.csv` | CSV индекс-файл (жесткая связка временных меток, координат и изображений). |

## 🕹️ Использование 3D-Viewer'а
Встроенный модуль `slam_viewer.py` открывается автоматически в конце работы скрипта.

* **Навигация по времени:** Используйте ползунок внизу окна, чтобы перемещаться по времени.
* **Просмотр кадров:** Слева будет динамически отображаться кадр с камеры для выбранной временной метки.
* **3D-сцена:** В 3D-окне агент (красная сфера) будет двигаться по пройденной траектории (оранжевая линия) внутри сгенерированного облака точек.

## 🔄 CI/CD & Оркестрация
Для удобства интеграции и поддержания высоких стандартов качества кода, проект поддерживает:

* **Makefile:** Вы можете использовать команду `make run DATA=floor_1` как удобную абстракцию над bash-скриптами.
* **Continuous Integration (CI):** Интеграция с GitHub Actions для автоматического тестирования сборки Docker-контейнера при каждом пуше в ветку `main`.
