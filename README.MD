ROS 2: Jazzy

Python 3.12+

Установленный и скомпилированный пакет OpenVINS.

Зависимости Python:

Bash
pip install open3d numpy pandas opencv-python PyQt5 scipy
📂 Структура проекта (Workspace Structure)
Для корректной работы автоматики убедитесь, что ваши данные (rosbag) лежат в директории data/ в корне вашего ros2_ws, строго следуя формату <floor>/<date>/<run>.

Plaintext
ros2_ws/
├── run_pipeline.sh               # Главный скрипт запуска (должен лежать в корне!)
├── data/                         # Директория с вашими данными
│   └── floor_1/                  # Пример: Этаж
│       └── 2025-05-05/           # Пример: Дата
│           └── run_1/            # Пример: Номер прогона
│               ├── rosbag/       # Исходный bag-файл (.db3 и metadata.yaml)
│               └── rosbag_pano/  # Сшитые панорамы (если используются)
├── src/
│   └── hilti-trimble-slam-challenge-2026/  # Исходный код этого репозитория
│       ├── build_index.py
│       ├── slam_viewer.py
│       └── challenge_tools_ros/  # ROS-пакет с логикой
└── dataset/                      # (Создается автоматически) Сюда сохраняются кадры и CSV
Примечание: папки build, install и log должны находиться только в корне ros2_ws, не создавайте их внутри src.

🛠 Сборка проекта
Перейдите в корень вашего ROS 2 workspace и соберите проект с флагом --symlink-install. Это позволит применять любые изменения в Python-скриптах моментально, без необходимости пересобирать пакет.

Bash
cd ~/ros2_ws
colcon build --symlink-install
chmod +x run_pipeline.sh
🎯 Запуск пайплайна
Главный скрипт сам определит свою директорию, поэтому его можно запускать из любого места в системе. Он принимает 3 аргумента: Этаж, Дата, Прогон.

Формат:

Bash
./run_pipeline.sh <FLOOR> <DATE> <RUN>
Пример:

Bash
cd ~/ros2_ws
./run_pipeline.sh floor_1 2025-05-05 run_1
⚙️ Что происходит:
Трансляция 2D-карты: Запускается map_server.launch.py для публикации базового плана этажа.

SLAM & Map Builder: Запускаются алгоритм OpenVINS и нода map_builder.py, которая в реальном времени собирает глобальное облако точек.

Обработка Rosbag: Запускается проигрывание данных со скоростью -r 0.5 (оптимально для стабильной передачи изображений и TF-деревьев).

Безопасная остановка (Graceful Shutdown): По завершении скрипт посылает сигнал SIGINT. Алгоритмы корректно завершают работу, сохраняя файл trajectory.txt и генерируя 3D-карту map_<run_name>.pcd в папке data/.

Синхронизация (build_index.py): Скрипт извлекает изображения из топика и привязывает их к координатам траектории, создавая файл sync_index.csv.

3D Визуализация (slam_viewer.py): Автоматически открывается интерактивный плеер на базе Open3D и PyQt5, в котором отображается карта, траектория и окно с синхронизированными панорамами.

⚠️ Примечания и советы (Troubleshooting)
Ручная остановка: Если вам нужно прервать пайплайн до окончания rosbag, нажмите Ctrl+C в терминале. Скрипт перехватит этот сигнал и корректно сохранит карту. Никогда не используйте жесткое закрытие терминала (или команду kill -9), иначе файлы .pcd и .txt не будут сгенерированы.

Поддержка WSL2 / Wayland: Если вы запускаете код в Windows Subsystem for Linux (WSL) или на Wayland, графический интерфейс (PyQt5/Open3D) может конфликтовать с драйверами. Скрипт автоматически применяет флаги LIBGL_ALWAYS_SOFTWARE=1 и WAYLAND_DISPLAY="" для обхода этой проблемы.

Черный экран в Open3D: Встроенный Map Builder собирает карту только с XYZ координатами. Наш визуализатор slam_viewer.py автоматически подкрашивает эти точки, центрирует вектор камеры (Up Vector) и рассчитывает масштаб для корректного отображения.

Переносимость кода: Все пути внутри скриптов построены относительно корня ros2_ws. Вы можете переименовывать или перемещать вашу рабочую папку в любое место — скрипты найдут нужные данные автоматически через os.environ.
